Deployment

Deployment is putting this stuff on an actual server. Real website. This, this is big boy stuff. But there's really only one big boy here, and that's me. So don't act like too much of a big boy, and use the dev server before you do anything wild.

Building Docker image#

From your own computer, you want to build the Docker image that you're gonna deploy on the server.

turtle docker:image --target dev --tag aegooby/<project>:dev-latest --prunedocker push aegooby/<project>:dev-latest

Connecting to the server#

Download the key file that I sent to you and put it in a location you'll remember. If I didn't send you a key file, maybe you shouldn't be reading this. To connect,

ssh -i <path to key file> root@dev.<domain>

Downloading Docker image#

On the server,

docker pull aegooby/<project>:dev-latestdocker tag aegooby/<project>:dev-latest dokku/<project>:latest

The magic part#

This part is really cool, assuming your changes weren't trash cyberterrorism that completely breaks the website.

dokku maintenance:on <project>dokku tags:deploy <project> latestdokku maintenance:off <project>
