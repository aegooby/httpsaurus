Codegen

The types in the GraphQL schema and in TypeScript files seem to be really similar to each other... if only there was a way to combine them. Well, while you've been wishing and dreaming with your head in the clouds, I'm here to tell you that there is a way to do that, thanks to yours truly. You're welcome. Well, I didn't make it, but I told you about it, so I'm still superior to you.

Running the code generator#

To generate the TypeScript types from the schema, all you have to do is run

turtle codegen

It's eerily close to replacing me and my entire job, so do me a favor and don't tell too many people that this exists. Actually by reading this, you're signing an NDA. This will hold up in court. Just trust me on that one.

Using the type definitions#

Generated code is written to the file graphql/types.ts. You can import the types from this file like this,

server/resolvers.ts
import type { Node } from "../graphql/types.ts";import { redisIndex } from "../graphql/types.ts";
note

Some imports from the types.ts file can't be done with import type. Always try to use import type first, and if you get an error, switch to normal import.
