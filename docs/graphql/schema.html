There are some requirements for a schema that aren't official in GraphQL but I'm making them official here.

Required interfaces#

Node#

The Node interface is a generic type for anything that has an ID. It should have an index and a tag on the id field for Global Object Identification. What's Global Object Identification? I just linked it twice, just click the damn link.

interface Node@index(name: "nodes", prefix: "nodes:") {    id: ID! @tag}

UserJwt#

This interface has the subset of fields of a user that should be stored in a JWT. See the authentication for info on what the fuck that is.

interface UserJWT {    id: ID!    # additional fields...}

Directives#

The schema has to declare these directives at the top:

# in graphql/schema.gqldirective @secret(name: String!, type: String!) on OBJECTdirective @index(name: String!, prefix: String!) on OBJECT | INTERFACEdirective @tag on FIELD_DEFINITION

@secret#

Arguments:

  • name: String!, the name of the field
  • type: String!, the TypeScript type of the field

The @secret directive adds a field to the Redis payload of an object that isn't exposed to the user, but is stored in the database. This is used for adding password fields to user types, for example, and unless you are an idiot you don't want random people being able to query the password.

Example:

# in graphql/schema.gqltype User implements Node & UserJWT@secret(name: "password", type: "string") {    id: ID!    email: String!}
// in server/resolvers.ts
import { RedisPayload, User } from "../graphql/types.ts";// ...const payload: RedisPayload["User"] = {  id: id,  email: args.email,  password: password,};await Redis.json.set(id, "$", JSON.stringify(payload));// ...

Here, the type RedisPayload["User"] will have the password field to be stored in the database, but the type User will not.

@index#

Arguments:

  • name: String!, the name of the index
  • prefix: String!, the prefix for Redis keys on which the index is applied

The @index directive adds a Redis index to all key-value pairs where the keys start with the prefix string. See this for information on how to automatically create the indexes in TypeScript.

Example:

# in graphql/schema.gqltype User implements Node & UserJWT@index(name: "users", prefix: "nodes:users:*:") {    id: ID!    email: String!}

This will add an index for all keys that start with "nodes:users:*:".

@tag#

The @tag directive adds a tagged field as part of a Redis index created with @index. This makes the field searchable.

caution

Using @tag in an object that does not have an @index results in an error.

Example:

# in graphql/schema.gqltype User implements Node & UserJWT@index(name: "users", prefix: "nodes:users:*:") {    id: ID!    email: String! @tag}
// in server/resolvers.ts
const escapedEmail = args.email.replaceAll("@", "\\@").replaceAll(".", "\\.");const search = await Redis.search.search("users", `@email:{${escapedEmail}}`);switch (typeof search) {  case "number":    throw new Error(`User with email ${args.email} does not exist`);  default:    break;}if (search.at(0) as number > 1 || search.length > 3) {  throw new Error(`More than one user found with email ${args.email}`);}const parsedSearch = search as [1, string, ["$", string]];const userInfo: RedisPayload["User"] = JSON.parse(  (parsedSearch.at(2) as ["$", string]).at(1) as string,);
