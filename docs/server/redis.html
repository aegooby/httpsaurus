Redis

Redis is used as the database, with the Search and JSON add-on modules. All data (except for rate-limit counters) is stored as JSON. Why? Because it's simpler. Stop asking so many questions.

danger

Don't use any Redis functions before creating the server using await Server.create(), since it handles conencting to the Redis instance.

JSON#

Get JSON data#

To get some data, use Redis.json.get(), with the key as the first argument, and the path as the second one. To get the whole user object, use "$" as the path. So this,

const data = JSON.parse(  await Redis.json.get(    "nodes:users:*:70af6899-189c-470c-8bd7-f14ffc3247b8",    "$",  ),);

Might return,

{  "id": "nodes:users:*:70af6899-189c-470c-8bd7-f14ffc3247b8",  "email": "mu@gabwe.com",  "password": "eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI..."}

But with a different path, we can do different shit. This,

const data = JSON.parse(  await Redis.json.get(    "nodes:users:*:70af6899-189c-470c-8bd7-f14ffc3247b8",    "$.email",  ),);

Might return,

"mu@gabwe.com"

Setting JSON data#

Setting data is just reversed. I bet you knew that. Okay don't be too proud of yourself, it was a really simple think to know.

const data = // ...  await Redis.json.set(    "nodes:users:*:70af6899-189c-470c-8bd7-f14ffc3247b8",    "$",    JSON.stringify(data),  );

Search#

Index creation is automated (see the GraphQL schema section), so creating indexes is as simple as,

Creating indexes#

// in server/daemon.tsx
import { redisIndex } from "../graphql/types.ts";for (const index of Object.values(redisIndex)) {  try {    await Redis.search.create(      index.name,      "JSON",      index.schemaFields,      index.parameters,    );  } catch {    undefined;  }}

Running a search#

To run a search, use Redis.search.search(). It will return 0 if there are no results, otherwise it will return an array with the format,

[  // number of search results, eg: 3  number,  // the key for a record, eg: "nodes:users:*:70af6899-189c-470c-8bd7-f14ffc3247b8"  key,  [    "$",    // the JSON data for the user as a string, eg: "{\"email\": \"mu@gabwe.com\"}"    json,  ],];

To search for a user by email, we do,

const escapedEmail = email.replaceAll("@", "\\@").replaceAll(".", "\\.");const search = await Redis.search.search("users", `@email:{${escapedEmail}}`);switch (typeof search) {  case "number":    throw new Error(`No user found with email ${email}`);  default:    break;}if (search.at(0) as number > 1 || search.length > 3) {  throw new Error(`More than one user found with email ${email}`);}const parsedSearch = search as [1, string, ["$", string]];const user = JSON.parse(  (parsedSearch.at(2) as ["$", string]).at(1) as string,);
